# CytoStudio Build Audit

This audit maps the current prototype to the requested prompt sequence. The app is now a runnable local-first product prototype, not merely the original blueprint catalog. Science-heavy and regulatory capabilities are represented as UI/state scaffolds until real reference data, engines, and validation suites are added.

## Current evidence

- Git repository initialized and committed: `b599ea8 Initial CytoStudio prototype`.
- App entrypoint: `index.html`.
- Application logic: `app.js`.
- Design system: `styles.css`.
- Original blueprint preserved: `blueprint-catalog.html`, `CytoStudio-Blueprint-and-Codex-Prompts.md`, and `prompts/`.
- Visual concept asset: `assets/design/cytostudio-workspace-concept.png`.
- FCS foundation: `fcs-core.js` plus `tests/fcs-core.test.cjs`.
- Worker import adapter: `fcs-import-worker.js`.
- Large-file import boundary: `docs/fcs-large-file-strategy.md`.
- Public FCS fixture provenance: `tests/fixtures/public/flowWorkspaceData/manifest.json`.
- Verification performed: JavaScript syntax check, local HTTP 200 check, Playwright desktop/mobile screenshots, Command-K check, tab-surface checks, and mobile overflow check.

## Prompt-by-prompt status

| Prompt | Status | Evidence / current behavior |
|---|---|---|
| 00 Vision / guardrails | Prototype complete | Three-pane app shell, state store, README architecture choice, light/dark support. Tauri/Rust/Arrow are documented next steps. |
| 01 Shell / navigation / design system | Prototype complete | Left sample rail, population tree, center canvas, inspector, toolbar, status bar, Command-K, responsive layout, persisted theme. Panel resizing/collapse is only lightly scaffolded. |
| 02 FCS import / experiment manager | Prototype partial | Drag-drop and file-picker imports now parse FCS 3.0/3.1 header/TEXT metadata, parameter labels, common numeric event data, keyword counts, sampled events, and spillover through `fcs-core.js`, with `fcs-import-worker.js` moving served-local parsing off the UI thread, streamed byte-level progress/cancel state, event-level parse progress, direct parser fallback when workers are unavailable, metadata-only preflight for oversized files, and sample-to-sample drag-drop grouping in the experiment browser. Fixture tests cover escaped delimiters, metadata-only parsing, max-event limits, parser progress callbacks, transforms, spillover, big-endian integer data, 8/16/32-bit integer widths, and a public Bioconductor `flowWorkspaceData` FCS3.0 DIVA fixture. Production memory mapping and broader public/vendor fixture coverage remain. |
| 03 Transforms / scaling | Prototype partial | Linear, log, arcsinh, logicle-like, and biexponential-alias transforms now live in shared tested code, expose per-axis floor/width/cofactor controls, auto-tune defaults from active parameter ranges/event previews, drive scatter/density normalization, and bin histograms from active event data. Full numerical parity with `flowCore`, canonical tick semantics, and formal reference validation remain. |
| 04 Plot canvas | Prototype complete | Canvas scatter, density, histogram, UMAP-style plots, legends, overlays, zoom gesture capture, publication-style chrome. GPU/datashader-scale rendering is not yet implemented. |
| 05 Manual gating | Prototype partial | Toolbar tools, gate creation, rectangle/polygon/lasso/ellipse/quadrant/interval overlays, labels, hierarchy updates, keyboard nudge feedback, and normalized gate geometry evaluation. Full vertex editing and robust boolean geometry remain. |

| 07 Statistics / table | Prototype partial | Inspector and table statistics now compute from gated event subsets: count, % parent, % total, median, mean, geometric mean, robust CV, percentiles, CSV export, clipboard copy, and a CD4/CD8 derived parameter. | 08 Compensation / spillover QC | Prototype partial | Compensation matrix state now parses embedded spillover keywords when present, matrix edits apply to the active compensated event stream, plots/statistics/gates refresh from compensated values, and the N-by-N QC grid plots real compensated channel pairs. | 09 Spectral unmixing | Prototype partial | Reference signatures, autofluorescence modeling, signature-quality warnings, rebuild-library action, and reviewable unmixing now create named unmixed fluorophore parameters available to plots and statistics. | 10 Templates / batch | Prototype partial | A dedicated Batch view now saves the active hierarchy as a template, applies it to the Treatment A group, pages through linked samples, stores sample-specific gate overrides with tailored badges, renders small-multiple gate review tiles, and recomputes combined group statistics into the table editor. Durable template files, real group assignment UX, high-volume recomputation, and full gate editing remain. |
| 11 High-dimensional / cluster explorer | Prototype partial | The High-Dim view now supports parameter selection, UMAP/t-SNE mode, FlowSOM/graph-style cluster assignment, progress/cancel state, marker coloring, sampled embedding rendering, cluster selection, cluster-by-marker heatmap, sample comparison history, and backgating a selected cluster into the population hierarchy. | 12 Figure editor | Prototype partial | The Figure view is now a page-canvas editor with positioned live plot tiles, text labels, arrows, picture-in-picture insets, layer selection, nudge/resize, snap/alignment/distribution controls, appearance controls for font/gate/tick style, reusable template save state, and browser-native SVG/PNG/TIFF/PDF proof exports. | 13 Pipeline / reports / export | Prototype partial | The Pipeline view now has a step-through action timeline, cursor controls, replay recording, report preview and HTML/PDF-proof generation, pipeline JSON export, gated FCS proof export, event CSV/Parquet proof export, statistics Excel-compatible export, and GatingML proof export. Binary FCS/Parquet/Excel writers, validated PDF generation, undo/redo state restoration, and GatingML conformance remain. |
| 14 Collaboration / workspace / interop | Prototype partial | The Share view now defines and exports a workspace JSON package with sample path/hash references, gates, plots, templates, figure settings, scientific settings, and pipeline history; supports local autosave/recovery, share-package export, shared-location sync toggle with change-log language, audit-log review, GatingML export/import proof, and FlowJo/Cytobank import proof actions. | 15 Onboarding / accessibility / polish | Prototype partial | First-run onboarding now offers bundled conventional and spectral sample launch paths, guided coach marks for gate -> population -> statistics, a keyboard shortcut sheet, Command-K help items, direct G/S/F/H shortcuts, high-contrast mode, ARIA-labeled help surfaces, clearer import errors, and responsive coach-mark behavior. Production authentication, validated signatures, retention enforcement, 21 CFR Part 11 validation, and lab SOP/legal review remain. |
